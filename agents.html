Agents

The problem

One conversation, one task, one thread. You can’t run research while writing while analyzing. Every task gets your full attention or none of it.

Agent Prime splits the work across specialized agents that each carry encoded expertise. Scout doesn’t write. Writer doesn’t analyze markets. Prime doesn’t do the work — it routes it. Specialization is how you scale output without scaling effort.

What gets encoded

11 agents organized by capability. Each is a standalone markdown file with its own methodology, quality gates, and context verification gate.

Scout
Scans sources for signals, evidence, and market data. Persists findings to structured signal files for downstream agents.
Synthesizer
Combines inputs into structured insight and theses. Turns raw signals into positions with evidence tiers and confidence levels.
Writer
Converts insight into publishable artifacts. Enforces voice rules, production order, and format constraints across every channel.
Industry Analyst
Deep structural mapping of industries. Nine-step methodology producing 80–90KB of layered analysis per run.
Investment Analyst
Investment-grade multi-lens analysis. Takes structural maps from Industry Analyst and runs valuation, moat assessment, and risk scoring.
Planner
Takes ideas through a 4-stage planning pipeline to a Build Handoff Spec. Plan before build — every time.
Builder
Executes Build Handoff Specs into deliverables. Receives the spec from Planner and ships without scope drift.
Connector
Builds strategic relationships and distributes work. Manages outreach sequences, follow-ups, and distribution across channels.
Experimenter
Runs experiments to validate claims empirically. Designs test conditions, tracks outcomes, updates confidence levels.
Clerk
Tracks commitments, deadlines, and staleness. Surfaces what’s overdue, what’s drifting, and what needs a decision.
Prime
Orchestrates, routes work, reviews quality, kills noise. The default agent when no specific capability is invoked.

Real example

Every agent prompt opens with a Context Verification Gate — a blocking checklist that runs before any output is produced. Here’s Scout’s:

agents/scout/prompt.md — Context Verification Gate
## Context Verification Gate
Before producing ANY output, verify access to:
- [ ] shared/context.md (identity + epistemic guardrails)
- [ ] shared/learnings.md (accumulated corrections)
- [ ] shared/registry.json (current work items)
- [ ] agents/scout/signals/ (prior signal scans)

If ANY file is missing, STOP and ask. Do not proceed
with degraded context.

This gate is non-negotiable. A 200-word output requires the same gate as a 4,000-word analysis. Skipping it because a task feels small is how you get generic output that ignores your thinking, voice, and prior corrections.

How to set it up

Each agent is a standalone markdown file. Start with one — Scout is good for research, Writer for content, Planner for breaking down projects. Each prompt follows the same structure:

Context Verification Gate
  → verify required files before producing anything

Role Definition
  → what this agent does and doesn't do

Methodology
  → step-by-step process, sequential not summarizable

Output Format
  → structure, naming conventions, where files land

Quality Gates
  → checks the agent runs before closing the task

Add more agents as your system grows. Each one compounds the value of the others — Scout signals feed Synthesizer theses, which feed Writer drafts, which Connector distributes. The pipeline self-reinforces once you have three or four agents running.
