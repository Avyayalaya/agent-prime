# Example: Startup Founder

> This is a fully worked example showing Agent Prime configured for a startup founder. Every file shows what a completed setup looks like.

## Persona

**Maya Chen** — Co-founder & CEO of Forge (Series A developer tools startup). Previously a senior engineer at Stripe. Building an AI-powered code review tool for enterprise engineering teams.

## What's Included

| File | Shows |
|------|-------|
| `context.md` | A filled-in identity, goals, voice, and intellectual breadth |
| `registry.json` | 6 real work items: theses, projects, tasks |
| `dispatch.md` | Active dispatch queue with 4 queued tasks |
| `config.json` | Goals mapped to agent combinations |

## How to Use This Example

1. **Study it** to understand what a completed setup looks like
2. **Copy and modify** if your domain is similar to startup founding
3. **Or just run the Onboarding Agent** (`@onboarder`) — it generates these files interactively

## Key Differences From the Template

Notice how Maya's setup differs from the blank template:
- Her goals (fundraising, product-market fit, thought leadership) map to different agent combinations than a corporate PM or researcher would use
- Her voice rules reflect startup communication norms (investor pitch clarity, not academic rigor)
- Her NDA constraints are about stealth-mode features, not employer IP
- Her registry items are startup-specific: competitive analysis, investor narrative, product thesis
